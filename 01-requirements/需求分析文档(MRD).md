# AI 智能求职辅导 Agent 平台 — 需求文档（MRD）

> 版本：v1.0　｜　日期：2026-06-16　｜　负责人：产品经理 / 全 Demo 负责人
> 范围：本期聚焦「简历智能润色」闭环（本地上传 → AI 改写润色 → 结果输出与下载），并完成前后端分离架构落地。

---

## 1. 背景与问题

求职者在投递前普遍面临三类痛点：

1. **简历与岗位错配**：简历内容偏流水账、缺少量化成果，与目标 JD 关键词不匹配，过不了 ATS 与 HR 初筛。
2. **表达不专业**：项目/经历描述口语化、动词弱、缺乏 STAR 结构，难以体现胜任力。
3. **修改成本高**：人工找人改简历贵且慢，通用 AI 工具又缺乏「岗位上下文」和「结构化输出」，改完格式乱、不可直接使用。

平台已在前期完成 MVP 验证：搭建了岗位 JD 与面经的 RAG 垂直知识库，跑通了意图识别路由与多轮对话状态机，交付了首字响应 < 1s 的高保真可交互 Demo。本期目标是将其**工程化落地为前后端分离架构**，并把最高频、最易变现的「简历润色」能力做成稳定、可上线的产品闭环。

## 2. 目标与非目标

### 2.1 产品目标
- 用户上传本地简历（PDF/Word），系统在可接受时延内输出一份**经 AI 润色改写、结构清晰、可直接使用**的简历。
- 支持「结合目标岗位 JD」进行定向润色，对齐关键词与能力项。
- 提供「修改对比」与「修改说明」，让用户理解每处改动的理由，可逐条接受/拒绝。
- 完成前后端分离架构落地，明确大模型 API 的数据交互边界，为后续能力（面试模拟、求职规划）扩展打底。

### 2.2 业务目标（衡量指标）
| 指标 | 目标值 |
|---|---|
| 简历解析成功率 | ≥ 98% |
| 润色任务首字响应（首 Token） | < 1s |
| 单份简历润色完整完成 | < 30s（流式输出） |
| 用户对润色结果满意（点赞率 / 采纳率） | ≥ 70% |
| 端到端任务成功率（上传→输出可下载） | ≥ 95% |

### 2.3 非目标（本期不做）
- 不做简历从 0 到 1 的「无中生有」生成（仅基于用户已有内容润色改写）。
- 不做精细化排版引擎 / 设计模板市场（本期仅提供 1~2 套标准模板导出）。
- 不做多人协作、版本分支管理。
- 不做面试模拟、Offer 谈判等其他 Agent 能力（仅预留架构扩展点）。

## 3. 目标用户与典型场景

- **应届/求职者**：投递前快速优化简历，对齐目标岗位。
- **被动求职者**：临时收到机会，需要快速把旧简历改得专业。

**核心用户故事**
> 作为求职者，我希望上传我现有的简历并选择目标岗位，系统能自动帮我把内容改写得更专业、更贴合岗位，并告诉我改了什么、为什么这么改，最后让我下载一份可以直接投递的简历。

## 4. 功能范围（本期）

| 编号 | 功能模块 | 说明 | 优先级 |
|---|---|---|---|
| F1 | 简历上传与解析 | 支持 PDF/DOCX 上传，解析为结构化字段 | P0 |
| F2 | 目标岗位输入 | 粘贴 JD 文本或选择岗位，作为润色上下文 | P0 |
| F3 | AI 润色改写 | 基于内容 + JD，流式输出润色后简历 | P0 |
| F4 | 修改对比与说明 | 原文 vs 润色后逐段对比，标注修改理由 | P0 |
| F5 | 逐条采纳/拒绝 | 用户可对每处修改接受或回退 | P1 |
| F6 | 结果导出 | 导出 PDF（采纳后的版本），套用标准模板 | P0 |
| F7 | 历史记录 | 保存润色任务，可回看与再次下载 | P1 |

详细的交互、字段、接口、状态流转见随附《PRD 详细设计》文件。
