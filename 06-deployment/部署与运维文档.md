# 部署与运维文档

> 阶段：部署上线　｜　版本：v1.0　｜　日期：2026-06-16
> 代码在 `04-development/`；本目录存放编排与运维资料。Dockerfile 随代码放在各自目录。

## 1. 环境变量（统一 LLM_ 前缀）

| 变量 | 说明 | 示例 |
|---|---|---|
| `LLM_API_KEY` | 中转站/官方密钥（仅服务端） | `sk-xxx` |
| `LLM_BASE_URL` | 中转站地址；官方留空 | `https://api.cisct.xyz` |
| `LLM_MODEL` | 主模型 | `claude-opus-4-8` |
| `LLM_FALLBACK_MODEL` | 降级模型，留空不降级 | 空 |
| `LLM_FILE_TTL_DAYS` | 上传文件保留天数 | `7` |
| `LLM_CORS_ORIGINS` | 允许的前端来源 | `http://localhost:5173` |

> 配置文件 `04-development/backend/.env`（已 gitignore，切勿提交）。

## 2. 本地开发部署

```bash
# 后端
cd 04-development/backend
python -m venv .venv && source .venv/Scripts/activate
pip install -r requirements.txt
cp .env.example .env        # 填 LLM_API_KEY / LLM_BASE_URL
uvicorn app.main:app --reload --port 8000

# 前端（另开终端）
cd 04-development/frontend
npm install
npm run dev                 # http://localhost:5173 （/api 代理到 8000）
```

访问 **http://localhost:5173**。

## 3. 容器化部署（Docker Compose）

```bash
cd 06-deployment
export LLM_API_KEY=sk-xxx
export LLM_BASE_URL=https://api.cisct.xyz
export LLM_MODEL=claude-opus-4-8
docker compose up --build
# 访问 http://localhost:8080
```

- `docker-compose.yml` 的构建上下文指向 `../04-development/backend` 与 `../04-development/frontend`。
- 前端镜像 = Nginx 托管构建产物 + 反代 `/api` 到后端（SSE 已关闭缓冲）。

## 4. CI/CD

见 `ci-example.yml`（GitHub Actions 样例）：后端装依赖跑 pytest、前端 `npm ci && npm run build`，主分支可加镜像构建与发布。接入时复制到仓库 `.github/workflows/ci.yml`。

## 5. 运维与可观测
- **健康检查**：`GET /api/health`。
- **日志**：uvicorn 访问日志 + 应用异常；生产建议接入集中式日志。
- **指标**：任务时延、成功率、模型 token 消耗、采纳率（见 `07-operations`）。
- **回滚**：镜像按版本打 tag；`docker compose` 切回上一个 tag 即回滚。

## 6. 安全运维
- 密钥仅 `.env`/环境变量，不入仓库、不入前端、不入日志明文。
- 上传文件按 TTL 自动清理（应用启动时清理过期文件）。
- 中转站密钥泄漏时：后台立即禁用/重置密钥并轮换。
